Executive Summary

Demo input showed USD 337.30 in known LLM/API spend across five features. A conservative first-pass savings target is USD 118.05, subject to quality testing before production changes.

The safest first changes are caching repeated context, capping non-critical output length, and moving low-risk internal tasks to cheaper routes after a small A/B test.

Top Cost Leaks

  • support-draft uses 56.5% of known spend on a low-risk internal route.
  • seo-tags has zero cache reuse despite repeated input patterns.
  • weekly-summary averages large outputs and should receive stricter output caps.

Routing Plan

  • Test cheaper or cached routes for low-risk support drafts and SEO tags.
  • Use a cheap draft plus stronger final check for medium-risk customer-visible summaries.
  • Keep high-risk checkout review on the stronger model until test data proves a safe fallback.

Implementation Checklist

  1. Add per-feature usage logging and daily budget alerts.
  2. Cap maximum output tokens by route instead of using one global cap.
  3. Add cache keys for repeated policy, brand, or context blocks.
  4. Run a small A/B test for low-risk draft and extraction tasks.
  5. Keep escalation rules for low confidence, policy-sensitive, or final customer output.
